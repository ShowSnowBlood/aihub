# Product

## Register

product

## Users

AI Hub operators and builders who need to collect, inspect, classify, review, and publish AI resources. They work in a backend console, usually checking source health, crawl volume, data quality, and pending review queues.

## Product Purpose

AI Hub aggregates AI news, GitHub AI projects, and external Skill resources from public sources such as skills.sh, GitHub repositories, Dify, n8n, and other agent/template marketplaces. Success means operators can see where data came from, trigger safe crawl jobs, diagnose failures, filter collected records, and move useful candidates into published community resources.

## Brand Personality

Precise, operational, trustworthy. The interface should feel like a serious data console, not a marketing page.

## Anti-references

Avoid decorative landing-page sections, oversized hero copy, vague AI slogans, nested cards, and visuals that make backend state harder to scan. Avoid scraping behavior that implies bypassing login, paywalls, captcha, or rate limits.

## Design Principles

1. Make source provenance obvious: every record should show its origin, collection method, and last run state.
2. Prefer dense but readable layouts: operators need to compare many sources and records quickly.
3. Separate actions from observation: monitoring, review, data source configuration, and crawl tools should be distinct.
4. Treat crawl safety as part of the product: show limits, modes, delays, checkpoints, and failure states.
5. Keep the primary path practical: trigger, monitor, inspect, clean, review, publish.

## Accessibility & Inclusion

Use readable contrast, keyboard-focusable controls, standard links and buttons, and reduced-motion-safe interactions. Data tables should remain usable on narrow screens through horizontal scrolling rather than broken columns.
